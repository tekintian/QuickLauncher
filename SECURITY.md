# 安全政策

QuickLauncher项目致力于为用户提供安全可靠的macOS应用程序。

## 🛡️ 安全承诺

- **隐私保护**: 不收集、不存储用户个人信息
- **代码安全**: 定期进行安全审计和漏洞扫描
- **透明开源**: 核心代码完全开源，接受社区监督
- **及时响应**: 对安全问题快速响应和修复

## 🔍 漏洞报告

### 报告范围
我们关注以下类型的安全问题：
- 权限提升漏洞
- 代码执行漏洞
- 信息泄露问题
- 拒绝服务攻击
- 跨站脚本(XSS)（如果涉及Web界面）
- 配置安全问题

### 报告方式
**请勿在公开Issue中报告安全问题！**

请通过以下方式联系：
- **邮箱**: tekintian@gmail.com
- **PGP密钥**: [点击获取PGP公钥](https://dev.tekin.cn/pgp)
- **加密消息**: 支持Signal/Telegram加密通讯

### 报告内容
请提供以下信息：
1. **漏洞描述**: 问题的详细说明
2. **影响范围**: 受影响的版本和系统
3. **复现步骤**: 详细的重现步骤
4. **攻击场景**: 可能的攻击方式
5. **修复建议**: 如有修复方案请提供

## ⏱️ 响应时间

- **确认接收**: 24小时内
- **初步评估**: 7个工作日内
- **修复发布**: 根据严重程度，7-30天内
- **安全公告**: 修复发布后7天内

## 🎯 安全评估标准

### 严重程度分级

#### 🔴 严重 (Critical)
- 可远程执行任意代码
- 可获取系统最高权限
- 可访问用户敏感数据

#### 🟠 高危 (High)
- 可提升本地权限
- 可绕过安全机制
- 可影响系统稳定性

#### 🟡 中危 (Medium)
- 信息泄露
- 拒绝服务
- 配置绕过

#### 🟢 低危 (Low)
- 信息泄露风险较低
- 影响范围有限
- 需要复杂条件才能利用

## 🔒 安全措施

### 开发安全
- **代码审查**: 所有代码变更经过安全审查
- **依赖管理**: 定期更新第三方依赖
- **静态分析**: 使用自动化工具扫描代码
- **签名验证**: 所有发布版本经过数字签名

### 构建安全
- **安全构建**: 在隔离环境中进行构建
- **完整性检查**: 发布包包含完整性校验
- **供应链安全**: 验证所有依赖组件来源
- **版本控制**: 安全的Git工作流程

### 运行时安全
- **权限最小化**: 仅请求必要的系统权限
- **沙箱隔离**: 使用macOS应用沙箱机制
- **代码签名**: 所有二进制文件经过Apple签名
- **自动更新**: 安全的自动更新机制

## 🛠️ 安全最佳实践

### 用户建议
1. **官方下载**: 仅从GitHub Releases下载
2. **验证签名**: 安装前验证代码签名
3. **及时更新**: 保持应用为最新版本
4. **权限审查**: 定期检查应用权限设置
5. **备份数据**: 定期备份重要数据

### 开发者建议
1. **安全编码**: 遵循Swift安全编码规范
2. **输入验证**: 严格验证所有用户输入
3. **错误处理**: 避免泄露敏感信息的错误消息
4. **日志安全**: 不在日志中记录敏感信息
5. **测试覆盖**: 包含安全测试用例
